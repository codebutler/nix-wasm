# Nix-built Guest Userspace — Plan 1: the system closure

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a **host-built Nix system closure** for the wasm32 NOMMU guest — `/etc` (nix.conf, profile, shells, TERM), a system profile (busybox + ncurses/terminfo), static passwd/group, and a busybox-init PID 1 with a generated inittab — generated by curated NixOS modules, with **no systemd/perl/python in the closure**.

**Architecture:** Reuse NixOS *module code* via `lib.evalModules` over a hand-picked module list + a stub module (Approach B, spike-verified in `spike/approach-b.nix`). Consume `config.system.build.etc` and `config.system.path`; add a static passwd/group generator and a busybox-init derivation; assemble a `wasm-system` toplevel in the boot layout the bootstrap (Plan 2) expects. No activation/systemd/eval-config machinery.

**Tech Stack:** Nix (flakes), the repo's existing wasm crossSystem (`wasm-cross.nix` + `deps-overlay.nix`), nixpkgs NixOS module tree, busybox/ncurses cross builds.

**Spec:** `docs/superpowers/specs/2026-06-16-nix-userspace-design.md`

**Scope:** This plan ends at a `nix build .#wasm-system` that produces the closure on the host. The `pc/vendor/linux-wasm/pc-init` bootstrap rewrite, initramfs repack, deploy, and in-guest boot/`sl`-renders verification are **Plan 2** (separate, depends on this closure + the cache).

## Conventions (read first)

- The nix daemon runs as root here. Run every build as its own command:
  `echo password | sudo -S nix --extra-experimental-features 'nix-command flakes' <args>`
  (`sudo -E` is ignored on this host; the sudo password is `password`.)
- Inspecting a built store path needs no sudo (`ls`/`cat` the `/nix/store/...` path).
- **Never** run two `nix` invocations concurrently (single SQLite eval cache → "database is busy").
- Closure litmus (used repeatedly): a path must NOT contain systemd/perl/python:
  `nix-store -qR <path> | grep -E 'systemd|perl-|python3' || echo CLEAN`
- Commit after each task. End commit messages with the Co-Authored-By trailer used in this repo.

```nix

  # cross compiler-rt with -DCMAKE_C_COMPILER_TARGET=wasm32-unknown-linux-musl,
  # which clang rejects. Apply the canonical wasm triple to compiler-rt for ALL
  # wasm consumers (not just curl/libarchive/boost), so busybox et al. build.
  # isWasm-guarded → native llvmPackages untouched (no rebuild cascade).
  fixCompilerRt = lp: lp.overrideScope (lf: lprev: {
    compiler-rt = lprev.compiler-rt.overrideAttrs (o: {
      cmakeFlags = builtins.map
        (f: builtins.replaceStrings [ "wasm32-unknown-linux-musl" ] [ "wasm32-unknown-unknown" ] f)
        (o.cmakeFlags or [ ]);
      postInstall = (o.postInstall or "") + ''
        ln -s $out/lib/*/libclang_rt.builtins-*.a $out/lib/libgcc.a 2>/dev/null || true
      '';
    });
  });
in
... # (keep the rest of the attrset; replace the old llvmPackages_21/llvmPackages lines with:)
  llvmPackages_21 = if isWasm then fixCompilerRt prev.llvmPackages_21 else prev.llvmPackages_21;
  llvmPackages = if isWasm then fixCompilerRt prev.llvmPackages_21 else prev.llvmPackages;
```

Note: `fixCompilerRt` must be defined in the overlay's `let` (top of `deps-overlay.nix`, beside `isWasm`/`whenWasm`). If Step 3 showed busybox resolving compiler-rt via `targetPackages.llvmPackages`, also add `targetPackages` handling per the error path — keep it `isWasm`-guarded.

- [ ] **Step 1: Create `userspace/system.nix`**

```nix
# Curated NixOS-module evaluation for the wasm32 NOMMU guest (spec Approach B).
# Reuses real NixOS module code (etc, system-path, nix, shells/system env,
# users-groups) to GENERATE config, with a stub module absorbing the
# systemd/initrd/activation options those modules write into. No systemd/perl.
{ nixpkgs, cross }:
let
  lib = cross.lib;
  modulesPath = nixpkgs + "/nixos/modules";
  result = lib.evalModules {
    specialArgs = { inherit lib; modulesPath = modulesPath; };
    modules = [
      (modulesPath + "/misc/assertions.nix")
      (modulesPath + "/system/build.nix")
      (modulesPath + "/system/etc/etc.nix")
      (modulesPath + "/config/system-path.nix")
      (modulesPath + "/config/users-groups.nix")
      (modulesPath + "/config/nix.nix")
      (modulesPath + "/config/shells-environment.nix")
      (modulesPath + "/config/system-environment.nix")

      # Stub: absorb writes from the curated modules whose DEFINING modules
      # (systemd, initrd, activation) we exclude. No behaviour.
      ({ lib, ... }: {
        options.boot.initrd.systemd.enable = lib.mkOption { type = lib.types.bool; default = false; };
        options.boot.initrd.systemd.users = lib.mkOption { type = lib.types.attrs; default = { }; };
        options.boot.initrd.systemd.groups = lib.mkOption { type = lib.types.attrs; default = { }; };
        options.boot.initrd.systemd.contents = lib.mkOption { type = lib.types.attrs; default = { }; };
        options.boot.initrd.systemd.storePaths = lib.mkOption { type = lib.types.listOf lib.types.anything; default = [ ]; };
        options.systemd.services = lib.mkOption { type = lib.types.attrs; default = { }; };
        options.systemd.tmpfiles.rules = lib.mkOption { type = lib.types.listOf lib.types.str; default = [ ]; };
        options.systemd.tmpfiles.settings = lib.mkOption { type = lib.types.attrs; default = { }; };
        options.system.activationScripts = lib.mkOption { type = lib.types.attrs; default = { }; };
        options.systemd.sysusers.enable = lib.mkOption { type = lib.types.bool; default = false; };
        options.services.userborn.enable = lib.mkOption { type = lib.types.bool; default = false; };
        options.security.wrapperDir = lib.mkOption { type = lib.types.path; default = "/run/wrappers/bin"; };
      })

      ({ config, lib, ... }: {
        options.nix.enable = lib.mkOption { type = lib.types.bool; default = true; };
        options.nix.package = lib.mkOption { type = lib.types.package; default = cross.nix; };
        config = {
          _module.args.pkgs = cross;
          _module.args.utils = import (modulesPath + "/../lib/utils.nix") {
            inherit lib config; pkgs = cross;
          };
        };
      })

      # Our config: a minimal NOMMU profile. Guest shell is busybox ash; bash is
      # native in this repo (deps-overlay maps it to buildPackages) so it is NOT
      # a guest binary and must not go in the profile.
      ({ lib, pkgs, ... }: {
        environment.systemPackages = lib.mkForce [
          pkgs.busybox    # init, ash (the guest shell), coreutils applets
          pkgs.ncurses    # terminfo (xterm-256color)
        ];
        environment.defaultPackages = lib.mkForce [ ];
        environment.variables.TERM = "xterm-256color";

        users.mutableUsers = false;
        users.users.root = {
          uid = 0;
          # single-user guest: passwordless autologin (Task 4 emits empty pw field)
          shell = "/bin/sh";   # busybox ash, linked by the bootstrap
        };

        nix.enable = true;
        nix.settings.experimental-features = [ "nix-command" "flakes" ];
        nix.settings.substituters = lib.mkForce [ "file:///nix-cache" ];
        nix.settings.require-sigs = false;
        nix.settings.sandbox = false;
      })
    ];
  };
in
{ config = result.config; }
```

## Task 4: Static passwd/group generator

`users-groups.nix` does not emit a static passwd (it relies on perl/userborn at activation). Generate passwd/group ourselves from `config.users`/`config.groups`, with an **empty** password field for passwordless autologin (matching today's `root::0:0:...`).

**Files:**
- Create: `userspace/passwd.nix`
- Modify: `flake.nix` (let-binding + `userspace-passwd` package for testing)

- [ ] **Step 1: Create `userspace/passwd.nix`**

```nix
# Static /etc/passwd + /etc/group for the wasm guest, derived from the module
# user/group model. Single-user guest: empty password field (no shadow) so getty
# autologin needs no credentials. Shells render as /bin/sh (busybox ash).
{ lib, pkgs, config }:
let
  usersList = lib.attrValues config.users.users;
  groupsList = lib.attrValues config.users.groups;
  gidOf = u:
    let g = config.users.groups.${u.group} or null;
    in if g != null && g.gid != null then g.gid else 0;
  passwdLine = u:
    "${u.name}::${toString u.uid}:${toString (gidOf u)}:" +
    "${u.description or ""}:${u.home or "/root"}:/bin/sh";
  groupLine = g:
    "${g.name}:x:${toString g.gid}:${lib.concatStringsSep "," (g.members or [])}";
  passwd = lib.concatMapStringsSep "\n" passwdLine
    (lib.filter (u: u.uid != null) usersList);
  group = lib.concatMapStringsSep "\n" groupLine
    (lib.filter (g: g.gid != null) groupsList);
in
{
  passwd = pkgs.writeText "passwd" (passwd + "\n");
  group = pkgs.writeText "group" (group + "\n");
}
```

## Task 6: busybox-init PID 1 + generated inittab

A tiny *real* PID 1: busybox `init` reading a Nix-generated inittab (getty→autologin per hvc console + syslogd), generated from Nix instead of `pc-init`'s heredoc.

**Files:**
- Create: `userspace/init.nix`
- Modify: `flake.nix` (let-binding + `wasm-init` package)

- [ ] **Step 1: Create `userspace/init.nix`**

```nix
# PID 1 for the wasm guest: busybox `init` + a generated inittab. One
# getty->autologin->ash chain per hvc console (must match HVC_WASM_NR_CONSOLES /
# js HVC_CONSOLES = 8), plus a file-backed syslogd. No systemd, no services.
{ lib, pkgs, nrConsoles ? 8 }:
let
  consoleLines = lib.concatMapStringsSep "\n"
    (i: "hvc${toString i}::respawn:/sbin/getty -L -i -n -l /bin/autologin 0 hvc${toString i} xterm-256color")
    (lib.range 0 (nrConsoles - 1));
  inittab = pkgs.writeText "inittab" ''
    ::respawn:/bin/sh -c '/sbin/syslogd -n -O /var/log/messages -s 16 -b 1; sleep 5'
    ${consoleLines}
  '';
in
pkgs.runCommand "wasm-init" { } ''
  mkdir -p $out/etc $out/bin
  cp ${inittab} $out/etc/inittab
  # autologin wrapper getty -l execs instead of /bin/login (single-user guest).
  cat > $out/bin/autologin <<'EOF'
  #!/bin/sh
  exec /bin/login -f root
  EOF
  chmod +x $out/bin/autologin
''
```

Note: the getty line now passes `xterm-256color` as the termtype arg, so even before `/etc/profile` runs `TERM` is correct (fixes the `vt102` default at the source).

# The guest system closure in boot layout. The Plan-2 bootstrap symlinks
# /etc -> $out/etc, /run/current-system/sw -> $out/sw, and execs $out/init.
{ pkgs, etc, systemPath, passwd, group, init }:
pkgs.runCommand "wasm-system" { } ''
  mkdir -p $out
  # Real /etc = module-generated etc + our static passwd/group. Preserve the
  # store symlinks inside etc (resolved in-guest, where /nix is mounted).
  cp -a ${etc}/etc $out/etc
  chmod -R u+w $out/etc
  cp ${passwd} $out/etc/passwd
  cp ${group} $out/etc/group
  ln -s ${systemPath} $out/sw
  cp -a ${init}/etc/inittab $out/etc/inittab
  cp -a ${init}/bin/autologin $out/etc/autologin   # bootstrap installs to /bin
  # init entrypoint: busybox init (the bootstrap execs $out/init)
  ln -s ${pkgs.busybox}/bin/busybox $out/init
