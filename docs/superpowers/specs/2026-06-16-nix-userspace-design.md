# Design: Nix-built guest userspace (kill the `pc-init` hacks)

**Date:** 2026-06-16
**Status:** approved (design) — implementation plan to follow
**Goal:** Replace the hand-written shell-script guest init (`pc-init`) + busybox
initramfs with a **Nix-built userspace generated by the NixOS module system**,
so the guest `/etc`, profile, users, `nix.conf`, `PATH`, `TERM`, and terminfo are
declarative package/module outputs instead of `printf`/`ln` lines.

---

## 1. Motivation

Three issues hit this session — sqlite WAL, `nix-env` command wiring, and
`TERM`/terminfo — all share one root cause: **the guest userspace is provisioned
by a hand-written shell script** (`pc/vendor/linux-wasm/pc-init`) on top of a
hand-built busybox initramfs. Each new need becomes another `printf` line. The
correct, shared layer is a Nix-built userspace where these are module-generated
config. This is `docs/plan-environment.md` Phase 2 ("the pc-linux userspace is
created by Nix"), made concrete.

What becomes declarative:

| Today (hack in `pc-init`) | After (module-generated) |
|---|---|
| `printf … > /etc/nix/nix.conf` | `config/nix.nix` → `nix.settings.*` |
| `echo 'root::…' > /etc/passwd` | Nix-generated from `config.users.users` |
| `export PATH=…/.nix-profile/bin` in `/etc/profile` | `config/shells-environment.nix` |
| `TERM` (busybox getty default `vt102`) | `environment.variables.TERM = "xterm-256color"` |
| terminfo (missing) | `ncurses` in `config.system.path` (corePackages) |
| `ln -sf /opt/bin/nix /bin/nix-env …` | system profile + a guest-tool shim |
| hand-built `inittab` | Nix-generated `inittab` for `wasm-init` |

## 2. Spike outcome (the design rests on verified evidence)

A spike (`spike/`, not committed) evaluated two approaches against the existing
wasm crossSystem (`wasm-cross.nix` + `deps-overlay.nix`, reproduced exactly —
identical `drvPath` to the flake's `cross.*`).

**Approach A — full `nixpkgs` `eval-config.nix` + `boot.isContainer = true`:
REJECTED.** It evaluates, but `config.system.path` dry-run = **261 derivations,
201 wasm cross-builds**, including `systemd-static-wasm32`, `perl-static-wasm32`,
`python3-static-wasm32`, and the full login/hardware stack (pam, shadow, sudo,
openssh, dbus, lvm2, cryptsetup, kmod…). Source: `config/system-path.nix`
`corePackages` + `defaultPackages`. `boot.systemdExecutable` only swaps the
stage-2 PID1 script; it does not remove systemd from the closure. `isContainer`
drops udev/kmod/hardware but leaves the entire default userland. Its `/etc`
doesn't even evaluate cross (`security/pam.nix` → `util-linux.lastlog`, dropped in
the cross build). A "trim it down" hybrid means `mkForce`-stubbing the whole tree
— reinventing B with more fragility.

**Approach B — curated `lib.evalModules`: ADOPTED.** A hand-picked module list
evaluates and cross-builds:
- `config.system.build.etc` dry-run = **6 trivial text derivations, 0 systemd/
  perl**. Realized for real → a store path with correct `/etc/nix/nix.conf`
  (`experimental-features = nix-command flakes`), `/etc/shells`, and
  `set-environment` exporting `TERM="xterm-256color"`.
- `config.system.path` (bash+busybox+ncurses) dry-run = **8 derivations, only
  `busybox` is real wasm payload**.

POC: `spike/approach-b.nix` (promote toward production). Reference (rejected):
`spike/approach-a.nix`. Cross set: `spike/cross-pkgs.nix`.

### Spike surprises folded into this design
1. **No free passwd/group/shadow.** `users-groups.nix` generates them at
   *activation* via perl (`update-users-groups.pl`) or `userborn`/
   `systemd-sysusers` — none available perl-less on NOMMU. → we generate a static
   `passwd`/`group` from `config.users.users` (correct for single autologin-root;
   `userborn` is mutable-state machinery we don't need).
2. **`bash` is intentionally native here** (`deps-overlay.nix` maps `bash`/
   `bashNonInteractive`/`runtimeShell` → `buildPackages`). `cross.bashInteractive`
   is NOT a guest binary. → guest shell = busybox `ash`.
3. **terminfo is fine.** Current `cross.ncurses` generates `share/terminfo`
   (xterm-256color incl.); the dangling symlink seen earlier was a stale *cached*
   ncurses. Fix = deliver ncurses in the profile + set `TERM`, not a tic rebuild.
   *(Re-verify against a fresh `cross.ncurses` build during implementation.)*
4. **`cross.busybox` currently fails** — the one common pkg pulling nixpkgs' bad
   cross `compiler-rt` (clang-rejected `wasm32-unknown-linux-musl` triple). Since
   busybox = init + shell + coreutils, extend the existing compiler-rt override to
   cover it (shared overlay fix, correct regardless).

## 3. Architecture

Three new units + two small fixes, each independently buildable/testable.

### 3.1 `wasm-system.nix` — the system config closure
`lib.evalModules` over a curated NixOS module list + a local **stub module**.

- **Modules:** `system/etc/etc.nix` (+ `system/build.nix`), `config/system-path.nix`,
  `config/nix.nix`, `config/shells-environment.nix`, `config/system-environment.nix`,
  `config/users-groups.nix`, `misc/assertions.nix`.
- **Stub module:** declares the systemd-shaped options the curated modules *write
  into* but whose defining modules we exclude — `boot.initrd.systemd.*`,
  `systemd.services`/`tmpfiles`, `system.activationScripts`,
  `systemd.sysusers.enable`, `services.userborn.enable`, `security.wrapperDir`,
  and a local 2-line `nix.enable`/`nix.package` (normally in
  `services/system/nix-daemon.nix`, which drags in the systemd module).
- **`nixpkgs.pkgs = cross`** (the wasm cross set) via `modules/misc/nixpkgs/read-only.nix`.
- **Config we set:** `environment.systemPackages` (busybox, ncurses, the guest
  `nix`), `environment.variables.TERM = "xterm-256color"`,
  `nix.settings.{experimental-features,substituters,require-sigs,sandbox,…}`,
  `users.users.root` (autologin, no password).
- **Outputs consumed:**
  - `config.system.build.etc` → guest `/etc`
  - `config.system.path` → guest system profile (`/run/current-system/sw`)
  - a Nix-generated `passwd`/`group` (see 3.2)

**Interface:** `import ./wasm-system.nix { inherit pkgs cross lib nixpkgs; }`
→ `{ etc; systemPath; passwd; group; }` (or a single `toplevel` dir assembling
them). Depends on: nixpkgs module tree + the wasm `cross` set. Does NOT depend on
systemd, perl, python, or activation.

### 3.2 Static passwd/group generation
A small function: `config.users.users` → `writeText "passwd"`/`"group"`. For the
single autologin-root guest this is `root::0:0:root:/root:/bin/sh` + `root:x:0:`
(matching today), but sourced from the module's user model so it stays in sync if
users are added. No shadow (no login passwords in a single-user guest).

### 3.3 `wasm-init.nix` — PID 1
busybox `init` + a Nix-generated `inittab`: one `getty -l autologin … hvcN`
respawn line per console (matching `HVC_WASM_NR_CONSOLES`) + the syslogd line, as
`pc-init` builds today, but generated from Nix. Built separately from the NixOS
activation/systemd machinery. `wasm-init` is what the bootstrap `exec`s.

### 3.4 `pc-init` → bootstrap (in the `pc` repo) — **Plan 2**

Shrinks to a thin bootstrap. The current Stage-2 (`exec $BB init` from a mounted
`/nix` busybox) already gestures at this; we make it the primary path. Fallback
to the existing busybox initramfs stays for a boot with no `/nix` store mounted.

**Boot-layout contract (what the bootstrap MUST do).** Plan 1 built the system
closure (`.#wasm-system`) with these assumptions — the entrypoint is **busybox
`init` reading `/etc/inittab`**, NOT a `wasm-init` wrapper:

1. mount proc/sys/dev/devpts + 9P (`/mnt/pc`, `/opt/bin`, `/nix-cache`, `/nix`).
2. symlink `/etc` → `$sys/etc` and `/run/current-system/sw` → `$sys/sw` (with
   `/nix` mounted, the store-symlinks inside etc — and `sw`/`init` — resolve).
3. `busybox --install` → creates `/bin/sh`, `/bin/login`, `/sbin/getty`,
   `/sbin/syslogd`, … (the applets inittab + autologin reference).
4. install `$sys/etc/autologin` → `/bin/autologin` (+x) **explicitly** — it is a
   custom script, not a busybox applet, so `--install` won't create it, yet
   inittab references `/bin/autologin` by absolute path.
5. create a writable `/var/log` (syslogd writes `/var/log/messages`).
6. `exec "$sys/init"` — the PATH (basename `init`), so busybox runs the `init`
   applet. Do NOT exec the resolved busybox target.

`nrConsoles = 8` is baked into the closure's inittab — keep kernel
`HVC_WASM_NR_CONSOLES` and host `HVC_CONSOLES` in lockstep.

Notes for Plan 2 (from the Plan-1 final review):
- **PATH is handled in-closure.** Plan 1 generates `/etc/profile` (the bash module
  that normally would is excluded) which exports `PATH=/root/.nix-profile/bin:
  /run/current-system/sw/bin:/bin:/sbin` and sources `/etc/set-environment`
  (TERM, TERMINFO_DIRS). busybox login sources `/etc/profile` for a login shell,
  so the bootstrap need not set PATH — it only must ensure the login chain is a
  *login* shell (the autologin → `login -f root` path already is).
- **The guest `nix` binary is NOT in `wasm-system`.** It's the separate
  `nix-wasm.nix` output, delivered to the guest store (today via `/opt/bin` +
  the cache). `nix-env -iA sl` needs it present independently of this closure.

### 3.5 Shared overlay fix
**Decision (locked): make the `compiler-rt` triple override general**, not
per-package. `deps-overlay.nix` applies the corrected `wasm32-unknown-unknown`
triple to the wasm `llvmPackages_21.compiler-rt` for *all* wasm consumers
(`isWasm`-guarded so native is untouched), so any package linking compiler-rt —
busybox included — gets the right triple. This replaces the current per-dep
application (curl/libarchive/boost) and matches the STATUS rule: shared
crossSystem fixes over per-package workarounds.

## 4. Data flow (boot)

```
pc kernel boots initramfs  →  /init (thin bootstrap, was pc-init)
  mount 9p: /mnt/pc, /opt/bin (tools), /nix-cache, /nix (store)
  ln -s  <store>/etc            /etc
  ln -s  <store>/sw             /run/current-system/sw   (PATH source)
  busybox --install ; install <store>/etc/autologin → /bin/autologin
  exec   <store>/init       (symlink → busybox; basename `init` → init applet;
                             reads /etc/inittab)
      → getty → autologin root → ash (login shell)
          sources /etc/profile  → PATH=/run/current-system/sw/bin:…
                                 → TERM=xterm-256color
          nix.conf, passwd, terminfo all already present from the store
```

The system closure (`etc` + `systemPath` + `wasm-init` + `passwd`) is built on
the **host** and substituted into the guest `/nix` store over 9P — same
substitute-only model as the package install path, and the caching design goal.

---

## 5. Error handling / fallbacks

- **No `/nix` store mounted** (e.g. tests, cache not registered): bootstrap falls
  through to the existing busybox initramfs userspace — a boot without the Nix
  system is unaffected (matches today's best-effort mounts).
- **Stub-module drift:** if a curated module starts writing a new systemd-shaped
  option, eval fails loudly with the missing-option error → add it to the stub.
  This is visible and contained, not silent.
- **Cross eval breakage:** any module pulling a cross-dropped attr (like A's
  `lastlog`) is caught at eval time; the curated list is small enough to audit.

---

## 6. Testing

1. **Host build:** `nix build` the system closure → assert the closure has **no**
   `systemd`/`perl`/`python` derivations (the spike's litmus); all substitutable.
2. **`/etc` contents:** nix.conf has the right substituters/experimental-features;
   `set-environment` exports `TERM=xterm-256color`; passwd/group correct.
3. **In-guest (pc kernel harness):** boot to a getty on hvc0; confirm `PATH`
   includes the profile; `nix --version`; `nix-env -iA sl`; and **`sl` actually
   renders** (closes the terminfo/TERM loop — the real acceptance test).
4. **Fallback:** boot with no `/nix` store registered → still reaches a shell.

---

## 7. Scope / non-goals (YAGNI)

- **No service manager.** The guest has no long-running services; systemd and any
  s6/runit-style supervisor are out. PID 1 only starts getty.
- **No `userborn`/perl activation.** Static passwd suffices for single-user.
- **No multi-user / shadow / PAM.** Single autologin root.
- **Not a general NixOS.** We reuse NixOS *module code* for config generation, not
  the distro/activation/eval-config machinery.
- **Kernel + guest-clang nixification** stay separate (later roadmap phases).

---

## 8. Risks / open items

- **Re-verify terminfo** against a fresh `cross.ncurses` (spike says fine; the
  cached path was stale).
- **busybox compiler-rt fix** must land first (blocks init+shell). Low risk —
  generalizing the existing override (§3.5). Watch for any native rebuild cascade
  (the `isWasm` guard should prevent it; verify a native pkg still substitutes).
- **One-shot `nix-env -iA` under NOMMU 9P memory pressure** is orthogonal
  (kernel-level) and out of scope here; the two-step `nix copy` + `nix-env`
  remains the reliable path until the kernel fix.
- **`pc` repo coupling:** the bootstrap change lives in `pc`; the system closure
  lives in `nix-wasm`. The interface is the store layout (`etc`, `sw`,
  `wasm-init`) the bootstrap expects.
